• Morning Notes

    Start with the shortlist, clear the obvious work, and leave the noisy tasks for later.

  • Design Review

    Check spacing, motion, and typography before adding another layer of visual detail.

  • Component Audit

    Confirm the states are readable, keyboard friendly, and resilient across narrow layouts.

  • Prototype Pass

    Keep the interaction simple first, then tighten the transitions once the behavior feels solid.

  • Content Queue

    Write real labels early so the interface is shaped by meaning instead of placeholder text.

  • Browser Check

    Test the edge cases, note the gaps, and avoid shipping a demo that only works on one machine.

  • Ship Notes

    Document the intent, the tradeoffs, and the next improvement while the context is still fresh.
